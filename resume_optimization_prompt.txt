================================================

6. Generates a full transparency report of every change made
7. Generates interview prep notes for any skill gaps or newly emphasized areas
8. Writes everything in natural, human-sounding language using the humanizer principles below

1. GitHub Profile Ingestion Service
------------------------------------------------------------
Create backend/app/services/github_ingestion.py

Use the public GitHub REST API (no auth token required for public repos):
- GET https://api.github.com/users/{username}/repos?per_page=100&sort=updated
- For each repo, fetch: name, description, language, topics, stargazers_count, forks_count,
  updated_at
- For the top 8 repos by recency/stars, also fetch:
    GET https://api.github.com/repos/{username}/{repo}/languages  (language breakdown)
    GET https://api.github.com/repos/{username}/{repo}/readme     (decode base64, truncate to
                                                                    1500 chars per README)
- If user provides a GitHub token (optional, from .env), pass it as Bearer header for higher
  rate limits
- Return a structured GitHubProfile dataclass:

    @dataclass
    class GitHubProfile:
        username: str
        repos: list[RepoSummary]        # name, description, languages dict, topics, readme_excerpt
        top_languages: dict[str, int]   # aggregated across all repos, sorted by frequency
        total_repos: int

- Use httpx async — do all repo detail fetches concurrently with asyncio.gather
- On rate limit (403/429), return whatever was collected so far — do not raise

2. LinkedIn Ingestion Service

LinkedIn blocks scraping aggressively. Use this approach:

3. Skill Inference Engine
------------------------------------------------------------
Create backend/app/services/skill_inference.py

This is pure Python logic — no Claude call here, save tokens.

Given a GitHubProfile and optional LinkedInProfile, infer a UserSkillProfile:

    @dataclass
    class UserSkillProfile:
        confirmed_skills: list[str]             # explicitly listed or clearly used
                                                # (language in 2+ repos)
        inferred_skills: list[str]              # reasonably inferable from project patterns
        project_evidence: dict[str, list[str]]  # skill -> list of repo names that support it
        seniority_signals: list[str]            # e.g. "has 3+ projects with READMEs"
        experience_years_estimate: Optional[int] # from LinkedIn date ranges if available
        domains: list[str]                      # e.g. "web", "data", "devops", "mobile"

Inference rules (implement as explicit logic, not AI):
- Language appearing in 3+ repos               -> confirmed skill
- Language appearing in 1-2 repos              -> inferred skill
- Repo topics containing "docker", "ci", "cd",
  "github-actions"                             -> infer DevOps familiarity
- README mentions "deployed", "production",
  "AWS", "GCP", "Azure"                        -> infer cloud exposure
- Repos with 5+ stars or 2+ forks              -> flag as notable projects
- LinkedIn skills list                         -> all go into confirmed_skills
- LinkedIn titles containing "senior", "lead",
  "architect"                                  -> add to seniority_signals

4. ATS Optimizer Service
------------------------------------------------------------
Read backend/app/services/ats_optimizer.py first — extend it, do not replace it.

Add a new method optimize_with_profile() that accepts:
- resume_text: str
- job_description: str

This method makes ONE Claude API call with a carefully structured prompt that does ALL of the
following in a single request:

OPTIMIZATION_PROMPT = """
You are an expert resume writer and ATS specialist. Complete all 4 tasks below in one response,
structured with these exact XML tags.

<job_analysis>
Extract from the job description:
- Required skills (explicit)
- Inferred preferred skills (reading between the lines)
- ATS keywords that must appear in the resume verbatim
- Seniority signals expected
</job_analysis>

<gap_analysis>
Compare job requirements against this user profile:
CONFIRMED SKILLS: {confirmed_skills}
INFERRED SKILLS: {inferred_skills}
PROJECT EVIDENCE: {project_evidence}
SENIORITY SIGNALS: {seniority_signals}

List:
- Skills the user clearly has (with evidence)
- Skills that can be reasonably inferred from their project work
- Skills the job wants that the user does not demonstrably have (honest gaps)
</gap_analysis>

<optimized_resume>
Rewrite the resume below against the job description. Rules:
- Weave in ATS keywords naturally — never keyword-stuff
- Use project evidence to support skill claims: if a skill appears in their GitHub work,
  reference that project context
- For skills the user has but has not articulated well, reframe their existing project
  descriptions to make those skills visible — use stronger, more precise technical language
  about what they actually built
- For minor skill gaps where the user's project work is adjacent (e.g. they used REST APIs
  but the job wants GraphQL), reframe the experience to highlight transferable depth — do not
  fabricate implementations that do not exist
- Every claim in the resume must be traceable to something real in their profile
- No em dashes. No buzzwords: spearheaded, leveraged, utilized, streamlined, robust, scalable,
  cutting-edge, best practices, passionate about, results-driven
- Sound like a smart human wrote it. Vary sentence structure. No parallel bullet fatigue.
- Quantify only where the project evidence supports a real number

RESUME:
{resume_text}
</optimized_resume>

<transparency_report>
Be specific and honest. List every change made:
- What was reworded and why (e.g. "Changed 'built a website' to 'developed a React SPA with
  client-side routing' — your GitHub repo confirms React usage")
- What keywords were added and where they were placed
- What was inferred from GitHub/LinkedIn vs explicitly stated
- Honest gap list: skills the job wants that are not in the user's profile at all — include
  study suggestions for each
- What the user should be ready to discuss in an interview given what is now on the resume
</transparency_report>

<interview_prep>
For every skill or technology now prominently featured on this resume, give:
- One likely interview question
- A one-sentence honest answer framework based on the user's actual project evidence
Focus on anything that was inferred or reframed — those are the interview risks.
</interview_prep>

JOB DESCRIPTION:
{job_description}
"""

Parse the Claude response by splitting on XML tags. Return a structured OptimizationResult:

    @dataclass
    class OptimizationResult:
        optimized_resume: str
        transparency_report: str
        interview_prep: str
        gap_analysis: str
        ats_keywords_added: list[str]
        linkedin_unavailable: bool
